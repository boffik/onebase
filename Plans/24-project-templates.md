# Этап 24 — Шаблоны проектов (`onebase init --template`)

**Статус:** ⬜ Не начато

## Контекст

`onebase init` создаёт пустой проект с одним примером. Шаблоны — готовые мини-конфигурации, которые ставятся одной командой и сразу запускаются. Это резко снижает порог входа и делает платформу «попробовать за 5 минут».

## UX

```bash
onebase init --template tasks   ./my-tasks
onebase init --template crm     ./my-crm
onebase init --template warehouse ./my-wh
onebase init --template finance ./my-fin

# Список доступных шаблонов:
onebase init --list-templates
```

## Шаблоны первой очереди

### `tasks` — мини-таск-трекер
- Справочник: `Проект`, `Исполнитель`
- Документ: `Задача` (Название, Проект, Исполнитель, Дата, Статус:enum, Приоритет:enum, Описание)
- Перечисление: `СтатусЗадачи` (Новая, В работе, На проверке, Закрыта), `Приоритет` (Низкий, Средний, Высокий)
- Отчёт: `ЗадачиПоИсполнителям`, `ПросроченныеЗадачи`
- Подсистемы: `Задачи`, `Администрирование`

### `crm` — мини-CRM
- Справочник: `Клиент`, `ВидДеятельности`
- Документ: `Сделка` (Клиент, Сумма, Статус:enum, МенеджерОтветственный, ДатаЗакрытия)
- Документ: `Платёж` (Клиент, Сделка, Сумма, Дата) с проведением
- Регистр: `Взаиморасчёты`
- Отчёт: `Воронка продаж`, `Долги клиентов`

### `warehouse` — склад
- Справочник: `Номенклатура` (иерархический), `Склад`, `Поставщик`
- Документ: `Поступление` с проведением
- Документ: `Реализация` с проведением, печатной формой, нумератором
- Регистр: `ОстаткиТоваров`
- Отчёт: `ОстаткиНаДату`, `ОборотыЗаПериод`

### `finance` — домашние финансы
- Справочник: `Счёт` (Наличные, Карта, ...), `Категория`
- Документ: `Операция` (Дата, Счёт, Категория, Сумма, ВидОперации:enum)
- Регистр: `ДенежныеСредства`
- Отчёт: `ОстаткиПоСчетам`, `РасходыПоКатегориям`
