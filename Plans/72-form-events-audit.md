# 72 — Аудит событий управляемых форм и до-проводка недостающих

## Контекст

Пользователь заметил, что при выборе из реквизита со списком значений не хватает
события, чтобы отобразить/подгрузить данные. Разбор показал две вещи:

1. Событийная модель **бэкенда богатая** (~25 типов в `internal/metadata/form_module.go`),
   а диспетчер `handleManagedFormEvent` универсален.
2. **Фронтенд** управляемой формы (`internal/ui/templates_managed.go`) исторически
   инициировал лишь `ПриОткрытии`, `ПриИзменении`, `Нажатие`. Многие объявленные
   события были «мёртвыми»: декларируются (и используются в примерах), но не
   вызываются. Параллельно существовал неиспользуемый `FormInterpreter`.

Решение: (A) дать декларативный реквизит «список значений» + событие при выборе;
(B) провести недостающие события и убрать мёртвый код.

## Карта событий (на момент аудита)

| Событие | Было | Стало |
|---|---|---|
| `ПриОткрытии` | ✅ клиент | ✅ |
| `ПриИзменении` | ✅ поля/грид | ✅ (+ `ПолеСписка`) |
| `Нажатие` | ✅ кнопки | ✅ |
| `ПриЧтенииНаСервере` | ✅ сервер (RLS) | ✅ |
| `ПередЗаписью`/`ПриЗаписи`/`ПослеЗаписи` | ❌ только определение | ✅ проведены в save |
| `ПриДобавленииСтроки`/`ПриУдаленииСтроки` | ❌ | ✅ на SlickGrid |
| `НачалоВыбора` | ❌ заглушка | ✅ динамический список `ПолеСписка` |
| `Выбор`/`ОбработкаВыбора` | 🟡 диалог подбора | 🟡 (без изменений) |
| `ПередЗакрытием`/`ПриЗакрытии`/`ПриАктивации`/`АвтоПодбор` | ❌ | ❌ (не трогали) |

## Что сделано

### Фаза A — реквизит «список значений» (`ПолеСписка`)
- `metadata.FormChoice` + поле `FormElement.Choices` (`internal/metadata/form_module.go`).
- `loadChoiceOptions` + ключ `ChoiceOptions` в контекст рендера
  (`internal/ui/handlers.go`, `handlers_managed.go`).
- Ветка рендера `ПолеСписка` (`<select>` из `choices`, выбор → `ПриИзменении`) в
  `internal/ui/templates_managed.go`.
- Пример: `examples/crm` форма «Сделка», поле «Валюта» + `ВалютаПриИзменении`.

### Фаза B — до-проводка событий
- **Write-lifecycle** (`internal/ui/form_server_events.go`, `handlers_entity.go`):
  `runPreSaveFormHooks` (ПередЗаписью/ПриЗаписи до Save, abortable, обогащает ссылки,
  сводит регистр ключей полей, возвращает ссылки к UUID) + `runAfterWriteFormHook`
  (ПослеЗаписи после Save с перезагруженным Объектом). Встроено в `submit`/`submitEdit`.
  Хелперы `canonicalizeFields`, `derefFields`, `pickObjectFormWithHook`,
  `renderObjectFormError`.
- **Детерминизм сериализации**: `serializeFieldsForEntity` теперь однозначно
  предпочитает мутацию хука (нижний регистр) исходному значению — раньше исход
  зависел от порядка обхода карты (флаки; задевало и события полей Фазы A).
- **События строк ТЧ**: SlickGrid дёргает `ПриДобавленииСтроки`/`ПриУдаленииСтроки`
  через `obFireRowEvent` (гейты `data-sg-rowadd`/`data-sg-rowdel`).
- **Динамический список**: билтин `ДобавитьЗначениеСписка` + поле ответа `choiceList`
  (`internal/ui/dynamic_choice.go`, `handlers_managed_events.go`); фронтенд —
  `obStartListChoice` (focus) + `applyChoiceList`.
- **Чистка**: удалён мёртвый `internal/dsl/interpreter/form_interpreter.go`
  (`FormInterpreter`/`FormContext` — никем не вызывались); обновлён устаревший
  комментарий в `form_module.go`.

## Ограничения / возможные продолжения
- Сообщения `Сообщить` из `ПриЗаписи`/`ПослеЗаписи` на успешном сохранении не
  показываются (редирект; как и сообщения самого `Save`) — можно завести flash.
- `НачалоВыбора` для `ПолеСписка` дёргается на `focus` нативного `<select>`
  (список обновляется к следующему открытию) — при желании заменить на picker-кнопку.
- Не проведены: фокус-события, `ПередЗакрытием`/`ПриЗакрытии`, `АвтоПодбор`.
