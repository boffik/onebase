# План автоматизации dev-цикла OneBase

## Цель
Создать удобный workflow для разработки: внести изменения → запустить скилл → увидеть результат в браузере.

## Текущее состояние
- ✅ `onebase dev` — сервер с hot reload для конфигурационных файлов (.os, .yaml)
- ✅ `devserver.Watch()` — файловый watcher с debounce 300ms
- ✅ Порт 8080 по умолчанию
- ❌ Нет автоматической пересборки при изменениях Go кода
- ❌ Нет автоматического перезапуска сервера после пересборки
- ❌ Нет автоматического открытия браузера

## Требуемый функционал

### 1. Полный hot reload (Go код + конфигурация)
**Проблема:** Текущий `dev` режим только перезагружает метаданные, не пересобирает бинарник.

**Решение:**
- Добавить watch для `**/*.go` файлов
- При изменении Go кода:
  1. Остановить текущий сервер
  2. Пересобрать `onebase.exe`
  3. Перезапустить сервер
  4. Сохранить состояние (подключения пользователей)

**Реализация:**
```
internal/cli/dev.go → добавить режим с пересборкой
internal/devserver/watcher.go → расширить для multi-dir watch
```

### 2. Автоматический запуск браузера
**Проблема:** Нужно вручную открывать `http://localhost:8080`

**Решение:**
- Добавить флаг `--open` или `--no-open` (по умолчанию `true`)
- Использовать `exec.Command("start", "url")` на Windows
- Открыть браузер только при первом запуске

### 3. Development CLI скилл
**Формула:** `onebase dev → edit → auto-reload → browser refresh`

**Команды:**
```bash
# Полный dev-цикл с пересборкой
onebase dev --project . --reload-binary

# Только конфигурация (быстро)
onebase dev --project . --reload-config

# С открытием браузера
onebase dev --open
```

## Этапы реализации

### Этап 1: Улучшение текущего dev режима
1. Добавить флаг `--open` в `devCmd`
2. Открыть браузер при старте если флаг=true
3. Улучшить вывод логов (что именно перезагрузилось)

### Этап 2: Binary hot reload
1. Создать wrapper процесс для управления жизненным циклом
2. Watch на `**/*.go` + watcher на конфиги
3. Graceful shutdown сервера при пересборке
4. Автоматический перезапуск

### Этап 3: CLI скилл для Claude
1. Создать `/dev-reload` команду/скилл
2. Логика:
   - Проверить изменения в Go файлах → пересобрать
   - Перезапустить сервер если нужно
   - Открыть/обновить браузер
3. Интеграция с Claude Code skill system

### Этап 4: Browser sync
1. WebSocket для browser reload
2. Автообновление страницы при изменениях
3. Сохранение состояния форм между перезагрузками

## Структура файлов

```
internal/cli/
  dev.go          → расширить флагами и логикой
  dev_reload.go   → новый файл для reload logic

internal/devserver/
  watcher.go      → расширить для multi-dir
  reloader.go     → новый файл для binary reload

scripts/
  dev.bat         → обёртка для быстрого запуска
```

## Приоритеты

**Высокий приоритет (сразу):**
- Флаг `--open` для автозапуска браузера
- Улучшенные логи что перезагрузилось

**Средний приоритет (следующая очередь):**
- Watch для Go файлов с пересборкой
- Graceful restart

**Низкий приоритет (будущее):**
- Browser sync через WebSocket
- Сохранение состояния между перезагрузками

## Альтернативы

**Быстрое решение (без изменений в коде):**
```bash
# PowerShell script для цикла разработки
while($true) {
  go build -o onebase.exe ./cmd/onebase
  ./onebase.exe dev
}
```

**Использование air (Go hot reload):**
- Установить `github.com/cosmtrek/air`
- Конфиг `.air.toml` для OneBase
- Минимальные изменения в коде

## Метрики успеха
- ⚡ Время от изменения кода до результата в браузере < 5 секунд
- 🔄 Автоматическая пересборка без команд пользователя
- 🌐 Автоматическое открытие URL при первом запуске
- 📢 Понятные логи что именно перезагрузилось
