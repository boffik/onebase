**Статус:** ⬜ Не начато

## Контекст

OneBase уже умеет **исходящую** сторону интеграций:
- HTTP-клиент в DSL (план 19) — звать внешние API из скриптов;
- **исходящие веб-хуки** спроектированы в плане 29 (⬜ не начато): декларативно
  «при проведении X → POST на URL».

Не хватает **входящей** половины: внешняя система (Telegram-бот, платёжка, маркетплейс,
n8n/Make) должна уметь **вызвать DSL-обработчик** OneBase по HTTP и создать/изменить
данные. Вместе с планом 29 это превращает OneBase в полноценный интеграционный хаб.

> Этот план — **входящие** хуки. Исходящие — план 29 (его стоит реализовать в паре).
> REST API сущностей (план 26) — это CRUD; здесь же — **произвольные именованные
> обработчики**, точка расширения под кастомную интеграцию.

## Синтаксис / UX

### Объявление эндпоинта в конфигурации

```yaml
# config/endpoints/telegram_in.yaml
name: TelegramВходящее
path: /hooks/telegram          # итоговый URL: /api/hooks/telegram
method: POST
auth: token                    # none | token | hmac
secret: "${env:TG_WEBHOOK_SECRET}"   # для token: заголовок X-Webhook-Token; для hmac: подпись тела
handler: telegram_in.os        # DSL-модуль с процедурой Обработать(Запрос, Ответ)
rate_limit: 60                 # запросов/мин на эндпоинт
```

### DSL-обработчик

```os
// src/endpoints/telegram_in.os
Процедура Обработать(Запрос, Ответ) Экспорт
    Данные = ПарсJSON(Запрос.Тело);
    Сообщение = Данные["message"]["text"];
    // ...создать документ/элемент справочника обычными средствами DSL
    Заявка = Документы.Заявка.Создать();
    Заявка.Комментарий = Сообщение;
    Заявка.Записать();
    Ответ.УстановитьКод(200);
    Ответ.Тело = ЗаписатьJSON(Структура("ok", Истина, "id", Заявка.Ссылка));
КонецПроцедуры
```

Объекты `Запрос` (метод, заголовки, query, тело, провалидированный JSON) и `Ответ`
(код, заголовки, тело) — новые DSL-объекты, инжектируются как `Движения`/`Запрос`.

## Хранилище / схема

Без новых таблиц на старте — эндпоинты грузятся из конфигурации (как обработки/отчёты).
Для аудита входящих вызовов — переиспользовать журнал регистрации или `_ai_audit`-подобную
таблицу `_endpoint_calls` (время, path, статус, длительность, IP) — опционально.

## Изменения в коде

| Файл | Изменение |
|---|---|
| `internal/metadata/endpoint.go` (новый) | модель `Endpoint` (path/method/auth/secret/handler/rate_limit) |
| `internal/project` | загрузка `config/endpoints/*.yaml` + модулей `src/endpoints/*.os` |
| `internal/runtime/registry.go` | `LoadEndpoints` + аксессоры (под RWMutex, как остальное) |
| `internal/dsl/interpreter/http_request.go` (новый) | DSL-объекты `Запрос`/`Ответ` |
| `internal/api/endpoints.go` (новый) | динамическая регистрация маршрутов `/api/hooks/*`; auth (token/HMAC); rate-limit; запуск handler через `Interp.Run` (в `execCtx` из плана 52) |
| `internal/configcheck` | валидация: уникальность path, существование handler-процедуры `Обработать`, корректность auth |

**Безопасность:**
- `auth: token` — постоянный заголовок-секрет; `auth: hmac` — подпись тела (для платёжек/Telegram);
- секрет из env (`${env:...}`), не из YAML напрямую — не утекает в экспорт конфигурации;
- rate-limit на эндпоинт (переиспользовать лимитер из плана 53);
- тело ограничено `MaxBytesReader`;
- handler исполняется в **песочнице** (план 57, этап 0) — входящий запрос недоверенный.

## Тесты

- `endpoints_test.go` — регистрация маршрута; `auth: token` отвергает неверный/пустой токен;
  `auth: hmac` проверяет подпись; rate-limit срабатывает.
- DSL `Запрос`/`Ответ` — парсинг JSON-тела, установка кода/заголовков/тела ответа.
- интеграционный: POST на `/api/hooks/...` создаёт документ, возвращает ожидаемый JSON.
- `configcheck` — дубль path / отсутствующая процедура `Обработать` → ошибка валидации.

## Verification

1. Объявить эндпоинт, послать `curl -XPOST /api/hooks/telegram` с валидным токеном и JSON —
   создаётся документ, ответ 200 с телом.
2. Неверный токен → 401; превышение rate-limit → 429.
3. `onebase check` ловит дубль path и несуществующий handler.
4. Связка с планом 29: исходящий webhook на проведение того же документа улетает на внешний URL.
