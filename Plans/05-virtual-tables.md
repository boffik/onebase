# Этап 5 — Виртуальные таблицы регистров (Остатки / Обороты / СрезПоследних)

## Контекст

Сейчас в `examples/simple-erp/reports/остаткитоваров.yaml` отчёт остатков пишется громоздко:

```
СУММА(ВЫБОР КОГДА вид_движения = "Приход" ТОГДА количество ИНАЧЕ -количество КОНЕЦ) КАК Количество
ИЗ РегистрНакопления.ТоварноеДвижение
СГРУППИРОВАТЬ ПО Номенклатура
```

В 1С это делается одной строкой: `ИЗ РегистрНакопления.ТоварноеДвижение.Остатки(&НаДату, ...)`. Без виртуальных таблиц отчёты перегружены и легко содержат ошибки в знаках. Эта фича добавляет в транслятор `internal/query` поддержку виртуальных таблиц регистров — самую важную абстракцию языка запросов 1С.

---

## Синтаксис

### Регистр накопления

```
// Остатки на дату (опционально с фильтром по измерениям)
ВЫБРАТЬ Номенклатура, КоличествоОстаток, СуммаОстаток
ИЗ РегистрНакопления.ТоварноеДвижение.Остатки(&НаДату, Склад = &Склад)

// Обороты за период
ВЫБРАТЬ Номенклатура, КоличествоПриход, КоличествоРасход
ИЗ РегистрНакопления.ТоварноеДвижение.Обороты(&Начало, &Конец)

// Совмещённая
ВЫБРАТЬ ... ИЗ РегистрНакопления.ТоварноеДвижение.ОстаткиИОбороты(&Начало, &Конец)
```

### Регистр сведений

```
// Срез последних на дату
ВЫБРАТЬ Курс ИЗ РегистрСведений.КурсыВалют.СрезПоследних(&Дата, Валюта = &Валюта)

// Срез первых
ВЫБРАТЬ Курс ИЗ РегистрСведений.КурсыВалют.СрезПервых(&Дата, Валюта = &Валюта)
```

### Колонки виртуальных таблиц

| Виртуальная таблица | Колонки |
|---|---|
| `.Остатки` | `<измерения>, <ресурс>Остаток` для каждого ресурса |
| `.Обороты` | `<измерения>, <ресурс>Приход, <ресурс>Расход, <ресурс>Оборот` |
| `.ОстаткиИОбороты` | объединение |
| `.СрезПоследних` / `.СрезПервых` | `<измерения>, <ресурсы>` |

## SQL-генерация

### Остатки

Для `Остатки(&НаДату, условие)` транслятор создаёт подзапрос:

```sql
(SELECT
    <dim1>, <dim2>, ...,
    SUM(CASE WHEN вид_движения = 'Приход' THEN <res1> ELSE -<res1> END) AS <res1>остаток,
    SUM(CASE WHEN вид_движения = 'Приход' THEN <res2> ELSE -<res2> END) AS <res2>остаток
 FROM рег_<имя>
 WHERE period <= $НаДату
   AND <условие фильтра>
 GROUP BY <dim1>, <dim2>, ...
 HAVING SUM(...) <> 0       -- скрывать нулевые остатки
) AS Остатки
```

### Обороты

```sql
(SELECT
    <dim1>, <dim2>, ...,
    SUM(CASE WHEN вид_движения = 'Приход' THEN <res> ELSE 0 END) AS <res>приход,
    SUM(CASE WHEN вид_движения = 'Расход' THEN <res> ELSE 0 END) AS <res>расход,
    SUM(CASE WHEN вид_движения = 'Приход' THEN <res> ELSE -<res> END) AS <res>оборот
 FROM рег_<имя>
 WHERE period BETWEEN $Начало AND $Конец
   AND <условие>
 GROUP BY <dim1>, <dim2>, ...
) AS Обороты
```

### СрезПоследних

```sql
(SELECT DISTINCT ON (<dim1>, <dim2>, ...) <dim1>, ..., <res1>, <res2>, ...
 FROM info_<имя>
 WHERE period <= $Дата
   AND <условие>
 ORDER BY <dim1>, ..., period DESC
) AS СрезПоследних
```

## Изменения в коде

### `internal/query/query.go`

- В лексере добавить распознавание точечной нотации после имени регистра (`РегистрНакопления.X.Остатки`).
- В парсере: расширить FROM-источник до варианта `<KindName>.<RegisterName>.<VirtualTable>(<args>)`.
- Метаданные регистров (известные ресурсы, измерения) нужны в момент компиляции — изменить сигнатуру:

```go
type CompileOpts struct {
    Params       map[string]any
    Registers    []*metadata.Register
    InfoRegs     []*metadata.InfoRegister
}

func Compile(src string, opts CompileOpts) (*Result, error)
```

- Сгенерировать subquery в FROM как описано в SQL-генерации.

### `internal/report/`

При выполнении отчёта прокидывать в `Compile` список регистров из `runtime.Registry`:

```go
result, err := query.Compile(report.Query, query.CompileOpts{
    Params:    params,
    Registers: reg.Registers(),
    InfoRegs:  reg.InfoRegisters(),
})
```

### `internal/dsl/interpreter/`

При создании `Новый Запрос()` (этап 14) передавать те же реестры регистров.
