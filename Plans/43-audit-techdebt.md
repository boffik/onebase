# Этап 43 — Техдолг и покрытие по итогам аудита

## Контекст

Внешний аудит (май 2026) и его верификация по реальному `go test -cover` выявили
зоны технического долга. Часть закрыта сразу (см. ниже), остальное вынесено в этот
план. С учётом того, что OneBase — **многопользовательская** система (малый бизнес,
несколько пользователей одновременно), приоритет отдан надёжности auth, конвертера и
корректной остановке сервера, а не «highload»-инфраструктуре.

### Уже сделано (коммиты `b1fd28f`, `65daaa0`)

- ✅ UTF-8 BOM убран из 41 `.go`-файла + CI-шаг падает при его возврате.
- ✅ `internal/auth` покрытие 7.9% → 61.3% (SQLite-тесты без build-тегов, идут в CI):
  аутентификация, bcrypt-смена пароля, сессии, «одна активная сессия», kick, роли
  через БД, middleware.
- ✅ `internal/converter` 0% → 63.5%, `parser1c` 0% → 24.1%: разбор v8.3-выгрузки,
  `parseType`, сквозной `Convert` → YAML.
- ✅ `coverage.out`, `*.test`, `*.log` в `.gitignore`.

Общее покрытие проекта: 28.2% → 29.6%.

### 43.2 — Полный graceful shutdown (приоритет 2, многопольз. критично)

HTTP-часть уже корректна (`SIGINT/SIGTERM` → `srv.Shutdown(ctx)` в `cli/run.go`,
`cli/dev.go`, `api/server.go`). Остаётся:
- дренаж активных регламентных заданий при остановке (`internal/scheduler`) —
  дождаться завершения или корректно прервать с записью в `_scheduled_runs`;
- атомарная запись бэкапа (temp-файл + rename), чтобы остановка не оставляла
  частичный `.obz`/`.sql.gz`.

### 43.3 — Единый стандарт логирования (приоритет 3)

`log/slog` уже используется точечно (3 места), но рядом ~19 `fmt.Println`/`log.Printf`.
- Завести единый логгер на `log/slog` с уровнями Error/Warn/Info/Debug.
- Контекстные поля: `request_id`, `entity`, `operation`.
- Заменить прямые `fmt.Println`/`log.Printf` в production-коде.

### 43.4 — Раскол монолитов (приоритет 4, по мере необходимости)

- Поведение не меняется — чистая перекладка ради скорости поддержки.
