Сделать так, чтобы конвертированная конфигурация **familyBg** (личные финансы) реально работала на OneBase. Это **sanity-check**, а не путь к продукту — следующая конфигурация снова даст мусор. ## Критерий успеха

1. `onebase serve` запускается без ошибок на директории familyBg.
2. Создание тестовых данных: 2 валюты, 2 кошелька, 2 проекта, 2 статьи, 1 курс валюты.
3. Проводятся все 6 типов документов: Поступление, Списание, Перемещение, Инвентаризация, ПоступлениеПлан, СписаниеПлан.
4. Отчёт «Остатки денег» возвращает суммы, совпадающие с ручной сверкой.
5. Отчёт «Доходы/расходы» возвращает корректные суммы по статьям.

## Объём (≤ 1 день)

### Очистка (5 минут)

Удалить 17 ложных каталогов из `catalogs/` (формы, модули, команды, обработки — попали туда конвертером). Полный список — в исходном плане Ч.1.1.

### Базовые YAML (30 минут)

- `enums/типы_кошельков.yaml` — один enum с 3 значениями
- `constants/constants.yaml` — `БазоваяВалюта` → `reference:Валюты`
- `catalogs/кошельки.yaml` — поле `Тип` → `enum:ТипыКошельков`
- `catalogs/статьи.yaml` — `hierarchical: true`

### Регистры накопления (30 минут)

5 регистров — `ДеньгиВКошельках`, `ДеньгиКПоступлению`, `ДеньгиКСписанию`, `ПоступлениеДенег`, `СписаниеДенег`. YAML по 5–10 строк каждый.

### Регистры сведений (15 минут)

2 регистра — `КурсыВалют` (периодический), `НастройкиПользователей`. Заполнить dimensions/resources.

### Документы — `posting: true` + numerator (15 минут)

Всем 6 документам добавить:
```yaml
posting: true
numerator:
  prefix: ""
  length: 6
  period: year
```

### Бизнес-логика — 6 файлов `.os` (3–4 часа)

Перенести «болванки» проведения из исходного плана (Ч.1.7). Все следуют паттерну OneBase: `Движения.<Регистр>.Очистить()` → `Дв = Движения.<Регистр>.Добавить()` → присвоение полей.

**Внимание:** перед написанием убедиться что в DSL OneBase есть:
- `Если ... Тогда ... КонецЕсли` ✓ (есть)
- `Для Каждого ... Цикл` ✓ (есть)
- `this.ТабЧасть` — итерируется ли в `Для Каждого`? **Проверить.**
- `this.Поле <> ""` для reference-поля — работает ли сравнение с пустой строкой? **Проверить.** Может потребоваться `Пустая()` (Plan C).

### Отчёты (30 минут)

4 отчёта в `reports/`:
- ОстаткиДенег — через `.Остатки(&НаДату)`
- ОборотноСальдовая — через `.ОстаткиИОбороты(&Начало, &Конец)`
- ДоходыРасходы — простая агрегация по регистру `ПоступлениеДенег`
- Планирование — простая агрегация по регистру `ДеньгиКПоступлению`

### Smoke-test (1 час)

Сценарий из критерия успеха. Зафиксировать **что не работает** — это вход в Plan C.
