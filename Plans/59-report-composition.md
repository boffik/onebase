## Контекст

Сейчас отчёты — это DSL-модуль `.rep.os` + вывод таблицы (план 03, далее). Дашборды и
виджеты уже есть (графики Chart.js, метрики, `homepage.yaml`). Чего не хватает бизнес-
пользователю — **интерактивной компоновки результата** без написания кода: группировки,
сворачивание, расшифровка ячейки, условное оформление, переключение «таблица/график».

Это **не полная СКД (1С)**, а её прагматичные 70 %: группировки + итоги + drill-down +
графики закрывают большинство задач малого бизнеса.

## Синтаксис / UX

### Декларативная компоновка в отчёте (YAML)

```yaml
# reports/продажи.report.yaml
name: ПродажиПоМенеджерам
title: Продажи по менеджерам
query: продажи.rep.os          # источник строк (как сейчас)
composition:
  group_by: [Менеджер, Период]    # иерархия группировок (сворачиваемая)
  measures:
    - field: Сумма
      agg: sum
      format: money
    - field: Количество
      agg: sum
  totals: true                    # итоги по группам и общий
  conditional:                    # условное оформление
    - when: "Сумма < 0"
      style: { color: red }
  drilldown:                      # расшифровка ячейки → список документов
    target: РеализацияТоваров
    by: Менеджер
  chart:
    type: bar                     # bar | line | pie
    x: Менеджер
    y: Сумма
```

### Поведение в UI
- Результат — сворачиваемое дерево групп с итогами на каждом уровне.
- Клик по ячейке-мере → расшифровка (drill-down) в список документов-источников.
- Переключатель «Таблица ⇄ График» (Chart.js, уже vendored/embedded).
- Условное оформление по правилам.
- Экспорт в Excel — уже есть (план 21), расширить на сгруппированный вид.

## Изменения в коде

| Файл | Изменение |
|---|---|
| `internal/report` / `internal/metadata` | модель `Composition` (group_by/measures/totals/conditional/drilldown/chart) в `Report` |
| `internal/report/compose.go` (новый) | агрегация плоских строк запроса → дерево групп с итогами (на сервере) |
| `internal/ui/handlers_reports.go` (после плана 55) | рендер сгруппированного результата + endpoint drill-down |
| `internal/ui/assets/report.js` (после плана 55) | сворачивание групп, переключатель таблица/график, Chart.js |
| `internal/ui/reportExcel` | экспорт сгруппированного вида |
| `internal/configcheck` | валидация: поля group_by/measures существуют в результате запроса; drilldown.target — известная сущность |

**Принцип:** агрегация и группировка — **на сервере** (по результату существующего
запроса), JS только рисует/сворачивает. Это переиспользует текущий конвейер отчётов и не
тянет тяжёлый клиентский pivot.

**Drill-down:** по правилу `drilldown` строим параметризованный переход в список
документов-источников с фильтром по значению группы (исп. существующие поиск/фильтры, план 23).

## Тесты

- `compose_test.go` — плоские строки → корректное дерево групп, итоги на уровнях и общий,
  несколько мер, разные agg (sum/count/avg/min/max).
- условное оформление: правило `when` помечает строки.
- `configcheck` — несуществующее поле в group_by/measure → ошибка.
- drill-down endpoint возвращает отфильтрованный список.
- Excel-экспорт сгруппированного отчёта открывается, итоги на месте.

## Verification

1. Отчёт с `group_by: [Менеджер, Период]` рендерится деревом с итогами; группы сворачиваются.
2. Клик по сумме менеджера → список его реализаций.
3. Переключатель «График» рисует bar-chart по тем же данным.
4. Отрицательная сумма подсвечена красным.
5. Экспорт в Excel сохраняет группировку и итоги.

## Эстимейт

- Модель `Composition` + серверная агрегация (`compose.go`): **2–3 дня**.
- UI: дерево/сворачивание/график/оформление: **3–4 дня**.
- Drill-down + Excel сгруппированного: **2 дня**.
- Валидация + тесты: **1.5 дня**.
- **Итого ≈ 1.5–2 недели.** Минимальный срез: группировки + итоги (без drill-down/графика) ≈ 4 дня.
