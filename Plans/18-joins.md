# Этап 18 — JOIN (соединения таблиц в языке запросов)

**Статус:** ✅ Реализовано

## Контекст

Текущий транслятор (`internal/query/query.go`) поддерживает только `UNION`/`ОБЪЕДИНИТЬ`. Без JOIN невозможно написать нетривиальный отчёт (например, продажи по контрагентам с расшифровкой по менеджерам). Это блокер для любого реального использования.

## Синтаксис

```
// Внутреннее соединение
ВЫБРАТЬ Прод.Номер, Клиент.Наименование, Прод.Сумма
ИЗ Документ.Реализация КАК Прод
    ВНУТРЕННЕЕ СОЕДИНЕНИЕ Справочник.Клиент КАК Клиент
    ПО Прод.Покупатель = Клиент.Ссылка

// Левое соединение
ВЫБРАТЬ Н.Наименование, СУММА(Д.Количество) КАК Итог
ИЗ Справочник.Номенклатура КАК Н
    ЛЕВОЕ СОЕДИНЕНИЕ РегистрНакопления.ОстаткиТоваров КАК Д
    ПО Н.Ссылка = Д.Номенклатура
СГРУППИРОВАТЬ ПО Н.Наименование
```

Поддерживаемые типы:
- `ВНУТРЕННЕЕ СОЕДИНЕНИЕ` / `INNER JOIN`
- `ЛЕВОЕ СОЕДИНЕНИЕ` / `LEFT JOIN`
- `ПРАВОЕ СОЕДИНЕНИЕ` / `RIGHT JOIN`
- `ПОЛНОЕ СОЕДИНЕНИЕ` / `FULL JOIN`

Специальное поле `Ссылка` (`Reference`) после точки → `id` (первичный ключ).

## Изменения в коде

1. Добавить в `kwMap`:
   - `ВНУТРЕННЕЕ` → `INNER`, `ЛЕВОЕ` → `LEFT`, `ПРАВОЕ` → `RIGHT`, `ПОЛНОЕ` → `FULL`
   - `СОЕДИНЕНИЕ` → `JOIN`
   - `INNER`, `LEFT`, `RIGHT`, `FULL`, `OUTER`, `JOIN`, `ON` — pass-through
   - `ПО` изменить с `BY` → `ON` (standalone `ПО` без СГРУППИРОВАТЬ/УПОРЯДОЧИТЬ всегда означает JOIN ON)

3. В обработчике `tDot` устанавливать `tr.prevWasDot = true`.

4. В обработчике `tIdent`: если `prevWasDot && upper == "ССЫЛКА"` → emit `id`.
