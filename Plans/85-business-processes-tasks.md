# План 85 - Бизнес-процессы и задачи

Дата проектирования: 2026-07-08.
Статус: ⬜ Проект (ранее деприоритизирован в `35-1c-feature-inventory.md` как
enterprise; вносится по запросу — для крупных внедрений это ядро, а не опция).

## Контекст

В инвентаре 1С-фич БизнесПроцессы/Задачи помечены «низкий приоритет». Но как
только OneBase используют для реального документооборота (согласование заявок,
маршрут «черновик → на согласовании → утверждён → проведён», делегирование), эта
подсистема становится центральной: без неё маршрутизация зашивается вручную в
`.os`-хуки и дублируется в каждой конфигурации.

Кирпичи для реализации уже есть: справочники/документы, регистр сведений
(состояние), DSL-хуки и модули менеджера, регламентные задания (план 17),
вебхуки (план 29), уведомления, RBAC + строковые политики (план 79).
Бизнес-процессы связывают их в декларативный маршрут.

## Цель

- декларативная **карта маршрута** (точки/переходы/условия) как объект
  конфигурации;
- объект **Задача** — адресованная пользователю/роли единица работы с состоянием,
  сроком, предметом (ссылка на документ);
- список «Мои задачи» в UI, выполнение/переадресация, автопереходы по условию;
- интеграция с проведением: «документ проводится только из точки Утверждён».

## Синтаксис / метаданные

Объект `bpm/<имя>.yaml` — карта маршрута:

```yaml
name: СогласованиеЗаявки
subject: Документ.Заявка          # предмет процесса
points:
  - { id: start,    kind: start }
  - { id: approve,  kind: task, assignee: { role: Руководитель }, name: Согласовать }
  - { id: revise,   kind: task, assignee: { field: Автор },       name: Доработать }
  - { id: done,     kind: end }
transitions:
  - { from: start,   to: approve }
  - { from: approve, to: done,   when: "Результат = 'Утверждено'" }
  - { from: approve, to: revise, when: "Результат = 'Отклонено'" }
  - { from: revise,  to: approve }
```

DSL для старта/движения из хуков документа:

```
БП = БизнесПроцессы.СогласованиеЗаявки.Старт(ЭтотОбъект);
```

Точки-задачи адресуются пользователю (`field:` реквизит), роли (`role:`) или
вычисляемому выражению; переходы срабатывают по `when`-условию на DSL.

## Хранилище / SQL

- `_bpm_instances` — экземпляр процесса: карта, предмет, текущая точка,
  состояние (`running`/`done`/`aborted`), автор, времена;
- `_bpm_tasks` — задачи: экземпляр, точка, исполнитель (kind+id), состояние
  (`open`/`done`/`redirected`), срок, результат;
- индексы под «мои открытые задачи» (исполнитель+состояние).

## Изменения в коде

- `internal/metadata/` — модель карты маршрута (`BusinessProcess`), валидация
  точек/переходов/адресации;
- `internal/bpm/` — движок: старт, генерация задач точки, выполнение задачи,
  вычисление переходов (через DSL-интерпретатор для `when`), автопродвижение;
- `internal/dsl/interpreter/` — объект `БизнесПроцессы.X.Старт/…` и `Задачи`;
- `internal/ui/` — «Мои задачи», карточка задачи (выполнить/переадресовать),
  визуализация маршрута; учёт RBAC + строковых политик (план 79: пользователь
  видит только адресованные ему/его области задачи);
- интеграция с `entityservice`: гейт проведения по точке маршрута (опц.);
- уведомления/вебхуки (планы 20/29) на назначение задачи.

## Тесты

- прохождение маршрута: старт → задача → условный переход → конец;
- адресация: задача попадает только нужному исполнителю (роль/реквизит/выражение);
- условия переходов на DSL, ветвление, возврат на доработку (цикл);
- права: чужие задачи не видны/не выполнимы (RLS);
- перезапуск/восстановление состояния экземпляра.

## Verification

1. Карта согласования заявки; создать заявку → у руководителя появилась задача.
2. «Утвердить» → процесс дошёл до конца, документ разрешено провести;
   «Отклонить» → задача автору на доработку, цикл повторяется.
3. Под чужой учёткой задача не видна.

## Эстимейт

2–3 недели по этапам: модель+валидация (2–3 дня), движок точек/переходов
(4–5), DSL-объекты (2), UI «Мои задачи»+маршрут (4–5), права/уведомления/
интеграция проведения (2–3), тесты (3). Начать с минимального среза
(start/task/end + условные переходы + «Мои задачи»), затем параллельные ветки,
сроки, эскалации.
