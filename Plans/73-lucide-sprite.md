# 73 — Полный Lucide через SVG-спрайт (подход B, продолжение плана 72)

## Проблема

План 72 отрисовал иконки разделов навигации подходом **A** — курируемый набор
инлайн-SVG Lucide в бинаре (`internal/ui/icons_data.go`, сейчас 44 иконки). Этого
хватает, но набор ограничен: чтобы добавить иконку, нужно регенерировать файл
данных. Когда пользователям регулярно нужны иконки вне набора и ручное расширение
списка начинает мешать — переходим на **B**: один спрайт со всем Lucide, где
работает любое имя.

Это не срочно. Браться, только когда ограниченность набора A станет реальным
неудобством (частые запросы «нет такой иконки»).

## Текущее состояние (что уже готово после плана 72)

Точка входа единая, поэтому миграция A→B локальна:

- **Хелпер.** `internal/ui/icons.go` — `LucideIcon(name) template.HTML` (обёртка
  `<svg class="lucide ob-icon" …>`, фолбэк `square`, синонимы `lucideAliases`,
  нормализация регистра/пробелов), `LucideNames`, `LucideIconsJSON`,
  `NormalizeIconName`. Данные — `internal/ui/icons_data.go` (сгенерированы из
  `lucide-icons/lucide`).
- **Рендер.** `internal/ui/templates.go` — `lucideIcon` в funcMap; вывод в
  `.subsys-bar`; CSS `.subsys-bar a .ob-icon`.
- **Конфигуратор.** `internal/launcher/configurator_tmpl.go` — превью у поля
  `icon`, `datalist` имён и live-preview-скрипт в `cfg-foot` (через
  `lucideIconsJSON`); нормализация при сохранении (`configurator.go`,
  `pages_handlers.go`).
- **Вендоринг ассетов (эталон для B).** `internal/webassets/assets.go` — паттерн
  `//go:embed` + `XxxHandler()`, монтаж под `/vendor/xxx/` через
  `http.StripPrefix` в ui- и launcher-серверах (Monaco/ECharts/SlickGrid/Quill).
  Тест-эталон — `internal/webassets/assets_test.go`.
- **Тесты.** `internal/ui/icons_test.go` — проверяют инлайн-пути (константы
  `sigShoppingCart`, `sigSquare`); при B их надо переориентировать на `<use>`.

## Решение: внешний SVG-спрайт

Все иконки — в одном файле `sprite.svg`, каждая как
`<symbol id="имя" viewBox="0 0 24 24">…</symbol>`. В месте показа — ссылка:

```html
<svg class="lucide ob-icon"><use href="/vendor/lucide/sprite.svg#shopping-cart"/></svg>
```

Браузер скачивает спрайт один раз и кэширует; дальше переиспользует символы по id.

## Где взять спрайт

- Собрать из `lucide-icons/lucide` `icons/*.svg`: обернуть внутренность каждого в
  `<symbol id="<имя>" viewBox="0 0 24 24">…</symbol>` (повторно использовать
  скрипт-генератор из плана 72 — он уже умеет вытаскивать внутреннюю разметку).
- Либо взять готовый спрайт из пакета `lucide-static`, если подойдёт формат
  (проверить наличие/структуру `sprite.svg`).

## Шаги

1. Сгенерировать `internal/webassets/lucide/sprite.svg` (весь Lucide как
   `<symbol>`). Зафиксировать обводку/атрибуты так, чтобы `currentColor`
   наследовался (как в инлайн-варианте A).
2. `internal/webassets/assets.go` — `//go:embed lucide/sprite.svg` +
   `LucideHandler()` (Content-Type `image/svg+xml`, длинный `Cache-Control`).
   Примонтировать `/vendor/lucide/` в **обоих** серверах (ui и launcher) — там же,
   где Monaco/ECharts/SlickGrid.
3. `internal/ui/icons.go` — переключить `LucideIcon` на вывод
   `<svg class="lucide ob-icon"><use href="/vendor/lucide/sprite.svg#<name>"/></svg>`.
   Сохранить семантику фолбэка/пустого имени:
   - пустое имя → пустая строка (как сейчас);
   - неизвестное имя → решить, как определять «нет такого символа». Варианты:
     (а) оставить список валидных имён (тогда A-данные не выбрасывать, держать как
     множество известных имён + фолбэк `square`); (б) принять, что любое имя
     уходит в `<use>` и битый id просто ничего не нарисует — тогда фолбэк не
     гарантирован. Рекомендуется (а): множество имён для фолбэка и для `datalist`.
4. `LucideIconsJSON` для конфигуратора — больше не вшивать разметку; в превью
   ссылаться на спрайт (тот же `<use>`), синонимы разрешать на клиенте.
5. CSS — `.ob-icon` уже есть; проверить, что размеры/цвет работают через `<use>`
   из внешнего файла (см. грабли ниже).
6. Доки `docs/features.md` — обновить, что доступно любое имя Lucide.

## Что учесть (грабли `<use>`)

- **Стилизация.** Обводка через `currentColor` наследуется; достучаться CSS-ом до
  отдельных частей символа из внешнего файла нельзя — нам не нужно (красим только
  обводку).
- **Первая отрисовка/кэш.** До загрузки спрайта возможен короткий «миг» без
  иконки на первой странице; после кэширования незаметно.
- **Совместимость.** Современные браузеры поддерживают внешний `<use>`; старые
  (IE/прежний Edge) — нет (исторически нужен был `svg4everybody`). Сейчас
  неактуально; именно из-за этого A надёжнее «из коробки».
- **Размер.** Весь Lucide — ~0.4–1 МБ в бинаре (embed). Митигировать сабсетом
  (класть в спрайт только реально используемые конфигурациями имена) — но это
  усложняет, теряется главный плюс «работает любое имя».

## Тесты

- `internal/webassets/assets_test.go` (по аналогии): `/vendor/lucide/sprite.svg`
  отдаётся с `image/svg+xml`; 404 на несуществующий путь.
- `internal/ui/icons_test.go` — переориентировать: `LucideIcon("shopping-cart")`
  содержит `<use … #shopping-cart`; пустое → пусто; (если выбран фолбэк-вариант а)
  неизвестное → `#square`.
- Рендер `.subsys-bar` — `<use … #shopping-cart>` присутствует.

## Вне области

- Загрузка произвольных пользовательских SVG.
- Иконки отдельных объектов (документов/справочников).
- Инлайн-спрайт (вставка всего спрайта в каждую страницу) — это «худшее из двух»
  по размеру HTML; не рассматриваем.
