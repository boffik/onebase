# Шаблон тела GitHub Release

Скопируй этот markdown в поле описания при создании Release на github.com — карточка релиза на onabase.site автоматически возьмёт из него красивое саммари и обложку.

---

## Как сайт превращает Release в карточку

Скрипт `releases.js` извлекает превью **в таком порядке** (первый сработавший — побеждает):

1. **HTML-маркер** `<!-- summary: текст -->` — самый надёжный, контроль 100%
2. **Секция с заголовком** из списка: `Что нового`, `Главное`, `Highlights`, `TL;DR`, `Summary`, `What's New`, `Overview`, `Кратко`, `Обзор`
3. **Fallback** — первый параграф любой секции, кроме `Установка / Installation / Requirements / Troubleshooting / Assets`, с пропуском строк-инструкций («Скачайте…», «1. Распакуйте…», `$ команда`)

**Обложка** работает по той же логике: первая картинка из любой *не-установочной* секции.

## Что попадёт на сайт

С шаблоном №2 карточка релиза покажет:

- **Обложка**: `your-cover.png` (первая картинка в `## 🎯 Главное`, картинки из `## Установка` игнорируются)
- **Заголовок**: тег релиза или Release Name, если задан
- **Саммари**: «DSL-редактор в браузере, hot reload без перезапуска, обновлённый конструктор отчётов.» (из `<!-- summary -->`-маркера)
- **Бейдж типа**: `minor` (вычисляется автоматически из semver, сравнивая с предыдущим стабильным релизом)
- **Дата публикации** — берётся из GitHub автоматически

На странице релиза (`release.html?tag=...`) — полный markdown с картинками, видео, alerts, code blocks и таблицами.

## Workflow

1. `git tag v0.X.Y && git push --tags`
2. На github.com → **Draft new release** → выбрать тег
3. **Release name**: оставить пустым (тогда покажется `v0.X.Y`) или ввести осмысленное название
4. **Describe this release**: вставить шаблон, заполнить `<!-- summary -->`, перетащить скриншоты прямо в текстовое поле (загрузятся в GitHub user-attachments)
5. **Publish release** — через ~15 минут (или сразу при hard refresh страницы) появится на сайте

## Советы

- **Скриншоты**: до 1–2 MB каждый, ужимать через [squoosh.app](https://squoosh.app/) до коммита
- **Видео**: лучше MP4 через `<video>` чем GIF — в 10× меньше; перетащить как файл, GitHub конвертирует автоматически
- **Эмодзи в заголовках** работают — `## 🎯 Главное` парсится корректно, эмодзи зачищается перед сравнением
- **GitHub Alerts** (`> [!NOTE]`, `> [!WARNING]`) рендерятся как красивые блоки в стиле GitHub
- **Тестовый превью**: после публикации открой `https://onabase.site/release.html?tag=v0.X.Y` и посмотри как выглядит. Если что-то не так — отредактируй release на GitHub, через 15 минут изменения подхватятся

## Кеш

Сайт кеширует список релизов в localStorage браузера на 15 минут. Чтобы увидеть свежий релиз сразу — `Ctrl+Shift+R` (hard reload) или в DevTools → Application → Local Storage → удалить ключи `onebase-releases-cache-v2:*`.
