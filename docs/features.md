## ИИ-помощник в конфигураторе
<!-- status: testing -->
<!-- since: build-273 -->
<!-- date: 2026-06-16 -->
<!-- issue: 75 -->

Конфигуратор научился использовать большие языковые модели прямо в процессе
разработки: **генерация каркаса** объекта по описанию словами (с предпросмотром
diff в staging), **объяснение ошибок** проверки конфигурации и **подсказка
запросов**. Сгенерированный код выполняется в песочнице с ограничениями.

**Как попробовать.** Откройте конфигуратор → ИИ-помощник, настройте ключ
(Gemini / Anthropic / OpenAI / совместимый) и сгенерируйте объект по описанию.

## Визуальный редактор макетов печатных форм
<!-- status: testing -->
<!-- since: build-274 -->
<!-- date: 2026-06-16 -->
<!-- issue: 73 -->

Печатные формы можно собирать визуально — расставлять элементы макета в редакторе,
без ручной правки YAML. Сейчас идёт UX-доводка: выделение, перетаскивание, свойства.

**Как попробовать.** Откройте печатную форму в конфигураторе и поработайте с макетом.

## ИИ-функции в языке (DSL)
<!-- status: testing -->
<!-- date: 2026-06-09 -->

Обращаться к моделям можно прямо из кода конфигурации: `ЗапросИИ`,
`ЗапросИИДжейсон`, `РаспознатьДокумент`, `РаспознатьИзображение`. Маршрутизация
«задача → цепочка моделей» — в ИИ-помощнике или через `llm` в `config/app.yaml`.

**Как попробовать.** Вызовите `ЗапросИИ("...")` из обработки и проверьте ответ.

## Бэкап и восстановление
<!-- status: stable -->
<!-- date: 2026-06-01 -->

Резервное копирование базы и восстановление из бэкапа — через CLI и интерфейс.
Переносите базу между PostgreSQL и SQLite одним `.obz`-файлом без потери нумерации,
пользователей и вложений.

**Как попробовать.** Сделайте бэкап рабочей базы и восстановите её в чистую.

## Печать в PDF и экспорт в Excel
<!-- status: stable -->
<!-- date: 2026-05-30 -->

Печатные формы выгружаются в PDF, списки и отчёты — в Excel (`.xlsx`).

**Как попробовать.** Откройте отчёт или печатную форму и выгрузите результат.

## Вложения к документам
<!-- status: stable -->
<!-- date: 2026-05-28 -->

К документам и элементам справочников можно прикреплять файлы (скан договора,
фото товара) — drag-and-drop на форме, файлы хранятся в файловой системе сервера.

**Как попробовать.** Перетащите файл на форму документа и сохраните.

## Email-уведомления (SMTP)
<!-- status: stable -->
<!-- date: 2026-05-25 -->

Отправка писем из DSL через SMTP — уведомления, отчёты на почту, рассылки.

**Как попробовать.** Настройте SMTP и отправьте письмо из обработки.

## HTTP-клиент в DSL
<!-- status: stable -->
<!-- date: 2026-05-22 -->

Объекты `HTTPСоединение` / `HTTPЗапрос` / `HTTPОтвет` с GET/POST/PUT/DELETE,
заголовками, телом и таймаутами — для интеграций с внешними API (курсы валют,
маркетплейсы, банк-клиент, Telegram).

**Как попробовать.** Запросите данные стороннего API и разберите JSON-ответ.

## JOIN в языке запросов
<!-- status: stable -->
<!-- date: 2026-05-20 -->

В запросах работают `ВНУТРЕННЕЕ` / `ЛЕВОЕ` / `ПРАВОЕ` / `ПОЛНОЕ СОЕДИНЕНИЕ ... ПО` —
для нетривиальных отчётов (продажи по контрагентам с группировкой по менеджерам).

**Как попробовать.** Соберите отчёт с соединением справочника и регистра.
